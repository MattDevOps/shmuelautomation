# Impeccable — Project Design Context

## Design Context

### Users
**Primary**: Shmuel — a Jerusalem real estate broker running Classic Jerusalem Realty.

- Older male professional, comfortable with desktop UIs but not a power user
- Speaks/reads English fluently for the admin; properties themselves often in Hebrew
- Uses this tool daily during business hours from a desk computer (light, well-lit office)
- This is HIS internal tool, not a SaaS pitched to a market — single-tenant, personal feel
- Cares about: getting work done quickly, not losing client/property data, looking professional to his peers
- Does NOT care about: trendy SaaS aesthetics, dashboards full of metrics, "AI-powered insights"

### Brand Personality
**Three words**: warm, authoritative, meticulous.

- *Warm*: Jerusalem-stone earthiness; this is a relationship business, not a transaction shop
- *Authoritative*: real estate is a high-trust domain; the interface should feel like a well-bound prospectus, not a startup demo
- *Meticulous*: every property field, every photo, every price is consequential — the design mirrors that care

Emotional goals: confidence, calm, "this is mine, not a generic tool."

### Aesthetic Direction
**Editorial / documentary**, with a Jerusalem-stone palette.

- Like a sophisticated print real-estate prospectus or an architectural monograph: generous whitespace, considered typography, photos treated as objects with captions and aspect-ratio integrity
- **Paper-warm light mode** (primary; this is a daytime tool). Dark mode supported but secondary
- **Jerusalem-stone palette** — paper-cream backgrounds, warm ink text, terracotta/clay accent that nods to the city's limestone and Mediterranean architecture
- **Asymmetric, left-aligned layouts** — never centered "hero" treatments; this is for working, not browsing
- **Hebrew + Latin coexist** — RTL must work in property text fields; typography pair must read well in both scripts

**Anti-references**:
- Generic Vercel/Linear "dark with purple gradient" — not us
- Material Design / shadcn defaults — too templated
- "AI dashboard with sparklines" — not us
- Anything that looks like Notion, Airtable, or HubSpot

### Design Principles

1. **Paper, not plastic** — surfaces feel printed, not extruded. Subtle warm borders, no shadows for depth, no glass effects. Flatness with intention.
2. **Density with breath** — admin tools must be efficient (Shmuel manages dozens of properties), but lines have generous height and prices/numbers have proper tabular alignment. Density without crowding.
3. **Photos are objects** — every property photo gets its proper aspect ratio, framing, and caption. Thumbnails aren't crops; they're considered miniatures.
4. **One accent, used sparingly** — terracotta for primary actions and active state only. Most of the interface is paper, ink, and edge-of-paper border.
5. **Type does the heavy lifting** — hierarchy comes from typography (size, weight, voice), not from boxes or color. A distinctive display serif paired with a refined sans body.
6. **Hebrew-first respect** — `dir="auto"` on free-text inputs; Hebrew property addresses render correctly without fighting the layout.

### Typography
- **Display**: Sentient (Fontshare) — warm contemporary serif, humanist, suits real-estate-prospectus voice
- **Body**: Switzer (Fontshare) — neogrotesque, six weights, good in dense tables
- **Numbers**: tabular-nums via font-feature-settings on price/size cells (no separate mono font)
- **Hebrew fallback**: system Hebrew fonts (SF Hebrew, Noto Sans Hebrew); design accommodates the slight optical mismatch

### Color (OKLCH)

- `--paper`: oklch(0.985 0.005 70)         — soft cream background
- `--paper-deep`: oklch(0.96 0.008 70)     — sunken paper for tables, code
- `--ink`: oklch(0.22 0.012 60)            — warm dark ink for text
- `--ink-soft`: oklch(0.46 0.01 60)        — secondary text

- `--clay`: oklch(0.55 0.13 35)            — terracotta accent

### Spacing (4pt scale, semantic names)
`--space-2xs: 4px; --space-xs: 8px; --space-sm: 12px; --space-md: 16px; --space-lg: 24px; --space-xl: 32px; --space-2xl: 48px; --space-3xl: 64px; --space-4xl: 96px`

### Banned (per impeccable rules)
- `border-left` / `border-right` greater than 1px as a colored stripe
- `background-clip: text` with gradient (gradient text)
- Pure `#000` or `#fff` — always tinted toward warm
- Generic AI-startup palette (cyan, neon-on-dark, purple-to-blue gradients)
- Cards-everywhere — only used where containment is meaningful
- Bounce/elastic easing
